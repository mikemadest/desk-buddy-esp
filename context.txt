Desk buddy project


Context: this is the code for an ESP Home project, 
based on yaml and c code (via lambda blocks). 
My purpose is to create a desk buddy project that react to touch and movement, 
make sounds via the passive buzzer and display eyes and mouth on screen. 

The hardware is an esp-s3 zero (they have one rgb w2812 led on board), 
one .96 mono oled screen 128x64 pixels, one vibration motor. 
Don't be too concerned by the hardware for now: 
it's working and the program is doing what it should.

The project uses main.yaml plus C++ includes under src/ (via esphome includes:).
Check https://esphome.io/guides/yaml/ and https://esphome.io/components/esphome/ (includes).

Focus files:
- main.yaml — hardware, automations, scripts, thin lambdas
- src/buddy_states.h — face/emotion constants
- src/buddy_faces.h / src/buddy_faces.cpp — face selection, transitions, procedural draw
- references/ — style inspiration for procedural faces (bigger eyes, blush, look/wink)
- assets/ — original bitmaps kept for reference only (not embedded)
Faces are fully procedural on a shared layout; neutral bridge between expression
families; heart eyes pulse by scaling procedural hearts. Shake uses intensity bands
(curious / surprised / dizzy @@→XX). Touch: short=pet, long=HUD, triple=face gallery.
Not everything from C++ may be available in lambdas; confirm with ESPHome API.
id() does not work inside included .cpp files — pass values from lambdas.

Commands:
1. validate the configuration: `python -m esphome config main.yaml`
2. compile the code: `python -m esphome compile main.yaml`
